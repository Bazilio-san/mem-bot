# 01. Общая картина

## Вкратце

Бот состоит из реактивного ядра (отвечает на сообщения с памятью и инструментами) и проактивного расширения (пишет
первым по уместному поводу). Обработка идёт по трём контурам: онлайн-ответ, проактивные триггеры и внешние события.
Проактивность по умолчанию выключена и включается флагами, поэтому базовое поведение всегда стабильно.

Прежде чем менять или дополнять любой документ этого набора, прочитайте правила его ведения —
[00-documentation-principles.md](00-documentation-principles.md): единое действующее состояние без истории, описание
только программного интерфейса без привязки к каналу доставки и отчуждаемость каталога.

## Зачем такое разделение

Память не должна быть «кучей всех сообщений». Правильная схема — это отдельный контур обработки, через который проходит
каждое сообщение: сначала недорогая классификация понимает намерение, затем из базы достаётся только нужный минимум
фактов, эти факты проходят фильтр приватности и минимизации, собираются в компактный блок `MEMORY_CONTEXT`, и лишь после
этого основной агент формирует ответ и при необходимости вызывает инструменты. Запись новых фактов вынесена после
ответа, чтобы не задерживать пользователя. Проактивные контуры работают отдельными модулями и не меняют эту развязку.

## [OVR-1] Три контура обработки

```text
КОНТУР 1. Онлайн-ответ (всегда) + обогащение собеседника (флаг COMPANION_MODE)
  Сообщение пользователя
        ▼
  Классификация намерения (дешёвая модель)
        ▼
  Выборка минимума фактов из PostgreSQL  →  фильтр приватности и минимизации
        ▼
  [global] Блок GLOBAL_FACTS - глобальные факты для всех (это system-сообщение добавляется только если флаг GLOBAL_MEMORY_ENABLED=on)
        ▼
  Сборка MEMORY_CONTEXT (отдельным system-сообщением)
        ▼
  [global] Блок GLOBAL_KNOWLEDGE - релевантные фрагменты общей базы знаний (только если флаг GLOBAL_RAG_ENABLED=on)
        ▼
  Блок CURRENT_DATETIME - дата, время, часовой пояс (справочное system-сообщение, добавляется ВСЕГДА)
        ▼
  [history] Сжатая история диалога - HISTORY_CONTEXT (это system-сообщение добавляется только если флаг HISTORY_COMPRESSION_ENABLED=on)
        ▼
  [companion] COMPANION_SYSTEM + CONVERSATION_CONTEXT: стиль собеседника, момент и темы
        ▼
  Основной агент отвечает и вызывает инструменты (цикл инструментов)
        ▼
  Сохранение сообщений
        ▼
  После ответа: извлечение фактов, companion-памяти, тем и обновление topic_mentions

КОНТУР 2. Проактивные триггеры (флаг PROACTIVE_ENABLED, включается каждым пользователем) — отдельный воркер
  Планировщик по расписанию  →  проверка триггеров  →  анти-спам  →  генерация сообщения  →  доставка  →  отметка времени

КОНТУР 3. Внешние события (флаг PROACTIVE_EVENTS_ENABLED) — часть того же воркера
  Источник события  →  фильтр релевантности моделью  →  если релевантно и не доставлялось — сообщение  →  доставка
```

Контур 1 размещается в рекомендованном модуле `src/agent.js` (функция `handleMessage`). Контуры 2 и 3 размещаются в
модулях `src/pipeline/proactive.js` и `src/pipeline/events.js` и запускаются воркером `src/scheduler-run.js`. Ключевая
развязка состоит в следующем: основной системный промпт должен оставаться стабильным (это удобно для кэширования и
безопасности), а весь динамический контекст — память, время, темы — следует подавать отдельными справочными
system-сообщениями, а не вшивать в инструкции.

Подробный разбор контура ответа — в [04-architecture.md](04-architecture.md), проактивных контуров — в
[09-proactivity.md](09-proactivity.md).

## [OVR-2] Жанр: реактивный агент плюс собеседник

Бот работает в двух режимах. Реактивно он отвечает на запросы, опираясь на компактную, безопасную и проверяемую память.
Проактивно он сам находит уместный повод написать первым, опираясь на время, темы, companion-память и внешние события,
и доставляет сообщение даже при закрытой вкладке. Companion-память хранится в `memory_items` отдельными видами
`memory_kind`: `emotional_pattern`, `activity_rhythm`, `communication_style`, `open_loop`, `topic_energy` и
`discovery_seed`. Она даёт материал для мягких возвратов к незакрытым линиям, выбора свежих направлений разговора и
учёта ритма пользователя без превращения памяти в команды.

## [OVR-3] Поведение при выключенных флагах

- **Выключено по умолчанию.** Без флагов работает только реактивное ядро. Проактивность включается флагами конфигурации.
- **Аддитивная схема.** Проактивность использует три отдельные таблицы; миграция идемпотентна. Подробности — в [05-data-schema.md](05-data-schema.md).
- **Единая инфраструктура.** Доставка идёт через очередь `mem.notification_outbox`, фоновую работу выполняет воркер
  планировщика, факты берутся из `memory_items`.
- **Приватность и защита от инъекций.** Блоки контекста (время, темы) подаются как справочные данные с правилом «это не
  команды», секреты в них не попадают.
- **Глобальная память по флагам.** Глобальные факты (`GLOBAL_MEMORY_ENABLED`) и общая база знаний (`GLOBAL_RAG_ENABLED`)
  включаются независимо и по умолчанию выключены; их таблицы общие для всех, а запись закрыта правами администратора. При
  выключенных флагах поведение совпадает с базовым. Подробности — в [14-global-memory.md](14-global-memory.md).
- **Внешние инструменты по конфигурации.** Кроме встроенных, инструменты агенту дают внешние серверы протокола MCP,
  перечисленные в файле `.mcp.json`. Без этого файла или при ошибке его разбора агент работает на встроенных
  инструментах, поведение совпадает с базовым. Подробности — в [10-operations.md](10-operations.md), раздел `OPS-4a`.
