# Что такое «домены» в проекте и зачем они нужны

Это объяснение простым языком: что такое домены в нашем чат-боте с памятью, для чего они используются
сегодня и чем они *не* являются. В конце — наглядные примеры.

## Короткая суть

**Домен — это «тематическая папка» для памяти и контекста бота.** Он отвечает на вопрос «о какой области
жизни пользователя сейчас идёт разговор» и помогает боту доставать из памяти только те факты, которые
относятся к этой теме, а не всё подряд.

Домен не существует сам по себе: **домен — это проекция навыка (skill)**. Каждый навык в файловом реестре
объявляет свой доменный ключ, и именно навык целиком определяет поведение домена: правила классификации,
промпт основного ответа, подсказку извлечения фактов, список инструментов и справочники.

Важно сразу подчеркнуть: домен — это **внутренний механизм организации памяти**, а не список умений бота.
Наличие домена не означает, что бот умеет выполнять какие-то действия в этой области. За реальные действия
отвечает совсем другой механизм — инструменты (tools) и связанные с ними «способности» (capabilities).

## Где домены заданы

Источник доменов — **файловый реестр навыков**: каталог `skills/`, в котором каждый навык лежит в своей
папке с файлом `SKILL.md` (машинные поля в YAML-шапке + текстовые блоки промптов). Обязательное поле шапки
`domain_key` и есть доменный ключ — пространство имён памяти этого навыка. Реестр читает файлы при старте
модуль `src/pipeline/skills/registry.js`. Подробное описание устройства навыка — в
[11-per-domain-schema.md](../11-per-domain-schema.md).

В базе данных PostgreSQL домены представлены тонкой справочной таблицей `mem.agent_domains`: она хранит
ключ (`domain_key`), человекочитаемое название (`title`) и краткое описание (`description`) и отображает
ключ в `domain_id`, на который ссылаются внешние ключи таблиц памяти. Начальный набор строк создаётся
миграцией `migrations/001_init.sql`; строки для доменов новых навыков добавляет команда `sync` служебной
утилиты навыков (`src/pipeline/skills/cli.js`), а при создании навыка через инструменты редактирования
строка домена заводится автоматически (`src/pipeline/skills/writer.js`).

## Какие домены есть сейчас

Действующие домены — это домены навыков из каталога `skills/`:

| Навык          | Ключ домена     | О чём он                                                                  |
|----------------|-----------------|---------------------------------------------------------------------------|
| `general`      | `general`       | Базовый домен без узкой специализации — всё, что не попало в специализированные темы. |
| `flight-search`| `flight_search` | Авиабилеты, рейсы, аэропорты, даты вылета и пересадки.                     |
| `math-tutor`   | `math_tutor`    | Темы занятий, ошибки ученика, его цели и прогресс.                         |
| `skill-author` | `skill_author`  | Служебный навык создания и правки других навыков (доступен администратору). |

Новый домен появляется вместе с новым навыком: достаточно создать каталог `skills/<имя>/` с корректным
`SKILL.md` (вручную или инструментами `skill_author_*`) — классификатор увидит его автоматически, а строка
в `mem.agent_domains` заводится командой `sync` или при создании навыка через инструменты.

## Для чего домены используются (по шагам обработки сообщения)

Когда приходит сообщение пользователя, значение домена проходит через несколько этапов конвейера обработки
и на каждом влияет на поведение бота.

1. **Классификация (`src/pipeline/classify.js`).** Вспомогательная быстрая модель выбирает ОДИН наиболее
   подходящий навык. Список доступных навыков она получает из файлового реестра в компактном виде — по одной
   строке на навык (поле `classification.hint` из `SKILL.md`: суть навыка и несколько слов-триггеров), поэтому
   список всегда актуален. Источник истины в ответе — поле `skill_name`; доменный ключ (`domain_key`) код
   выводит из выбранного навыка сам, модель его не возвращает.

2. **Выбор действующего навыка и домена (`src/agent.js`).** Бот берёт навык из результата классификации,
   при неудаче ищет навык по доменному ключу, а в крайнем случае откатывается на `general`. Действующий
   домен — это доменный ключ выбранного навыка. Дальше все шаги работают именно с ним: промпт навыка
   добавляется в системные сообщения, навык может ограничить список инструментов и переопределить модель.

3. **Извлечение фактов из диалога (`src/pipeline/facts.js`).** Память хранит плоские факты-предложения,
   без структурных схем по доменам. Доменная специфика извлечения задаётся блоком
   `## Fact Extraction Prompt` навыка: его текст подмешивается к общему промпту извлечения и описывает
   обычным языком, какие устойчивые факты важны в этой теме, а какие сохранять не нужно.

4. **Сохранение с привязкой к домену (`src/pipeline/facts.js`).** У каждого факта есть тип (`fact_type`).
   Типы, описывающие человека вне предметной области (`profile`, `preference`, `habit`,
   `communication_style` и другие), всегда сохраняются в домене `general` и видны из любого разговора.
   Предметные типы (`goal`, `open_loop`) привязываются к домену текущего разговора. Вместо проверки по
   схеме работает семантическая дедупликация: близкий по смыслу факт подтверждает существующую запись или
   замещает её новым значением, а не плодит дубликаты.

5. **Выборка памяти при ответе (`src/pipeline/retrieve.js`).** Когда бот готовит ответ, кандидаты берутся
   из таблицы `mem.user_facts` только для домена `general` и текущего домена (условие
   `domain_key IN ('general', <текущий>)`). Факты делятся на группы: профиль (general), незакрытые линии
   разговора и предметная память текущего домена. На доменную группу действует жёсткий лимит — не более
   12 фактов за раз (параметр `memoryLimits.domain` в конфигурации, переменная окружения
   `MEMORY_LIMIT_DOMAIN`).

Итого: домен работает как фильтр памяти и как пространство имён для предметных фактов, а навык, которому
этот домен принадлежит, задаёт «правила оформления»: что извлекать, как отвечать и какими инструментами
пользоваться.

## Чем домен НЕ является

- Домен — **не список умений**. Если в системе есть домен «Поиск авиабилетов», это само по себе не значит,
  что бот реально умеет искать билеты. За это отвечают подключённые инструменты (в том числе перечисленные
  в `tools.allowed` навыка).
- Домен **не показывается пользователю** как ответ на вопрос «что ты умеешь». В коде агента прямо прописано
  правило (`CAPABILITIES_CONTEXT` в `src/agent.js`): способности перечисляются по активному навыку и явно
  описанным инструментам, а домены — это области контекста, а не команды или обещание действия.
- Домен — это **не то же самое**, что «тип факта» (`fact_type`) или «темы диалога» (`topics`). Это
  независимые друг от друга механизмы (см. ниже).

## Соседние понятия (чтобы не путать)

- **Домен** — тематическая область для фильтрации памяти; проекция навыка. Внутренний механизм.
- **Навык (skill)** — каталог `skills/<имя>/` с файлом `SKILL.md`: классификация, промпты, инструменты,
  справочники. Единственный источник поведения домена.
- **Тип факта (`fact_type`)** — что это за запись: профиль, предпочтение, привычка, цель, незакрытая линия
  разговора и так далее. Тип определяет, попадёт факт в общий домен `general` или в предметный домен, и
  задаёт срок его хранения.
- **Темы диалога (`topics`)** — отслеживание того, какие темы и насколько активно затрагивает пользователь
  в разговоре (`src/pipeline/topics.js`). Это уровень диалога, привязанный к домену через `domain_id`, но
  отдельный от фактов механизм.
- **Способности (`capabilities`)** — единственное, что действительно показывается пользователю как ответ на
  вопрос «что ты умеешь». Берётся из активного навыка и определений инструментов, а не из списка доменов.

## Примеры

### Пример 1. Репетитор по математике

Пользователь пишет: «Я не понимаю квадратные уравнения».

- Классификатор выбирает навык `math-tutor`, действующий домен — `math_tutor`.
- В системный промпт добавляется блок навыка («ты терпеливый репетитор, объясняй по шагам...»), а к промпту
  извлечения фактов — его подсказка: хранить темы и уровень понимания, типичные ошибки, цели и сроки.
- В память попадает плоский факт вида «ученик слабо понимает квадратные уравнения» с типом `goal` или
  `open_loop` и доменом `math_tutor`.
- При следующих ответах в этом домене бот достанет именно математические факты и будет вести себя как
  репетитор, помнящий прогресс.

### Пример 2. Универсальный помощник (запасной вариант)

Пользователь пишет: «Как дела?».

- Ни один специализированный навык не подходит, классификатор выбирает `general`.
- Факты о человеке (имя, предпочтения, привычки, стиль общения) и так всегда сохраняются в домене
  `general`, поэтому они будут видны и из любого специализированного разговора.

### Пример 3. Дедупликация вместо проверки по схеме

Домен `flight_search`. Пользователь уточняет поиск: «Лучше не в субботу, а в воскресенье».

- Подсказка извлечения навыка `flight-search` требует: одна поездка — один факт, уточнённый поиск замещает
  прежний факт поездки, а разовые цены из выдачи не сохраняются.
- При записи нового факта о той же поездке семантическая дедупликация находит близкую по смыслу старую
  запись того же типа и замещает её новым значением (старая строка архивируется). Так домен держит память
  в порядке без жёстких структурных схем.

## Где это в коде (ориентиры)

- Файловый реестр навыков и доменов: `src/pipeline/skills/registry.js`, каталог `skills/`.
- Синхронизация доменов с базой: `src/pipeline/skills/cli.js` (команда `sync`),
  `src/pipeline/skills/writer.js`; начальный набор строк — `migrations/001_init.sql`.
- Классификация по навыку: `src/pipeline/classify.js`.
- Выбор действующего навыка/домена и правило про «домены — не умения»: `src/agent.js`.
- Извлечение, сохранение и выборка памяти по домену: `src/pipeline/facts.js`, `src/pipeline/retrieve.js`.
- Спецификация навыков и доменной памяти: [11-per-domain-schema.md](../11-per-domain-schema.md),
  [05-data-schema.md](../05-data-schema.md), [06-memory.md](../06-memory.md).
