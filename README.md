## Обзор

MEM BOT — это агентская система: языковая модель, многоуровенвая память и набор инструментов.  
Она вызывает инструменты, управляет памятью, ставит напоминания, читает защищённые записи — пока не решит задачу;
набор инструментов расширяется внешними серверами MCP, а набор навыков - кастомными навыками.

Бот помнит контекст между разговорами (профиль, предметные факты, защищённые данные, напоминания), но в каждый запрос
подставляет лишь небольшой релевантный и безопасный срез памяти. Базовое ядро реактивно; необязательный проактивный
режим позволяет писать первым, не превращаясь в спам.  

Бот ориентирован на собеседника как на человека: запоминает, кто вы и как вам удобнее общаться, подстраивает стиль и бережёт личные данные.

## Что умеет бот

### 🧠 Память — как она устроена

Главный принцип памяти: **в базе можно хранить много, но в каждый запрос к модели подставляется лишь малый,
релевантный и безопасный срез** (как правило, от 10 до 30 фактов). Так бот помнит контекст между сессиями, но не
зачитывает всю историю и не переполняет окно модели. Память делится на глобальную (общая для всех) и личную
(привязана к собеседнику):

| Вид памяти | Область | Что хранит |
|------------|---------|------------|
| **Глобальные факты** | общая | Всегда-актуальные факты для всех, подмешиваются в каждый запрос к модели. |
| **База знаний (RAG)** | общая | Документы; ищутся по смыслу (эмбеддинги), в запрос попадают только релевантные фрагменты. |
| **Профиль** | личная | Устойчивые факты о собеседнике и его стиле общения. |
| **Диалог** | личная | Факты текущего разговора. |
| **Предметные факты** | личная | Сведения по теме активного навыка (домена). |
| **Защищённые данные** | личная | Пароли, ключи, личное — зашифровано (AES-256-GCM), только с явного согласия. |
| **Напоминания** | личная | Дела на будущее, которые бот доставит по расписанию. |

Несколько уточнений к таблице:

- **Защищённые данные** в рассуждениях модели фигурируют только обезличенной сводкой, без раскрытия полного значения.
- **Глобальную память** наполняет и чистит только администратор; оба её механизма по умолчанию выключены и независимы.
  Глобальные факты — всегда-включённый минимум для всех, а база знаний (RAG) подтягивается выборочно под вопрос, поэтому
  может быть сколь угодно большой, не раздувая каждый запрос.
- **История диалога** автоматически сворачивается в краткий дайджест, чтобы не раздувать контекст.

### 🧩 Настраиваемые навыки — поведение под задачу

Поведение бота задаётся **навыками** — каталогами `skills/<name>/`. Один навык — это законченный домен: свой
namespace памяти, промпт ответа, правила извлечения фактов, закрытая схема предметных данных и набор инструментов. Бот
сам определяет по сообщению, какой навык включить. В комплекте идут:

| Навык | Назначение |
|-------|------------|
| `general` | Универсальный ассистент по умолчанию. |
| `flight-search` | Поиск авиабилетов. |
| `math-tutor` | Репетитор по математике. |
| `skill-author` | Создание и редактирование навыков прямо из диалога (только администратору). |

Навык `skill-author` делает систему самонастраиваемой: новыми доменами бота можно расширять прямо из диалога, не трогая
код. Пока создание и редактирование навыков доступно только администратору — это ограничение текущей версии.

### 💬 Общается в двух каналах

С ботом можно говорить из терминала (быстрый способ всё проверить) или через полноценного бота в Telegram. В Telegram
ответ печатается на ваших глазах по мере генерации, и видно, какими инструментами бот сейчас пользуется. Чтобы решать
задачи, модель сама вызывает нужные инструменты — управление памятью, напоминания, защищённые записи, настройки и
работу с навыками.

### 🎙️ Понимает голос и отвечает голосом

Бот распознаёт голосовые сообщения и при желании отвечает озвученной речью с выбором голоса. Оба направления по
умолчанию выключены и включаются настройками.

### ⏰ Напоминает и по-дружески пишет первым

Можно попросить бота напомнить о деле — разово или регулярно; за доставку отвечает отдельный планировщик, который
переживает перезапуски. В режиме собеседника бот может написать первым по уместному поводу: при затянувшемся молчании,
с утренним приветствием, по движению к цели или при внешнем событии. Чтобы это не превращалось в спам, действуют
настраиваемые пороги и суточные лимиты; по умолчанию режим выключен.

### 🔌 Внешние инструменты

Бот умеет подключать внешние инструменты по протоколу Model Context Protocol (MCP), расширяя набор доступных модели
действий без изменения собственного кода. Включается флагами конфигурации.

## Установка и быстрый старт

### Требования

- Node.js версии 20 или новее.
- PostgreSQL 16 с расширениями `pgvector` (векторный поиск) и `pgcrypto` (шифрование защищённых данных).
- Заполненный файл `.env` (за основу возьмите `.env.example`): строка подключения к базе и ключ доступа к языковой
  модели (`OPENAI_API_KEY`). Адрес `OPENAI_BASE_URL` нужен только при работе через OpenAI-совместимый прокси.

### Шаги

```bash
npm install          # установка зависимостей
cp .env.example .env # затем впишите в .env строку подключения к БД и ключ модели
npm run migrate      # создаёт базу, схему mem и применяет все миграции из migrations/
npm run chat         # интерактивный чат в терминале — самый быстрый способ проверить бота
```

Для напоминаний по расписанию запустите отдельный воркер планировщика командой `npm run scheduler`, а для работы в
Telegram — команду `npm run telegram` (нужен заполненный `TELEGRAM_API_KEY`).

## Использование

### Чат в терминале

Команда `npm run chat` открывает интерактивный диалог. Доступны служебные команды:

| Команда | Назначение |
|---------|------------|
| `/domain <ключ>` | Сменить активный навык-домен (`general`, `flight-search`, `math-tutor`, `skill-author`). |
| `/fact <текст>` | Принудительно сохранить факт в память. |
| `/kb <текст>` | Добавить запись в общую базу знаний (RAG). |
| `/tick` | Прогнать планировщик напоминаний вручную, не дожидаясь воркера. |
| `/exit` | Выйти из чата. |

### Бот в Telegram

Команда `npm run telegram` запускает бота. Он поддерживает потоковый вывод черновика ответа по мере генерации,
показывает статус вызываемых инструментов, принимает голосовые сообщения и при включённом голосовом выводе отвечает
озвученным сообщением. Полная процедура запуска, проверки и отладки описана в
[docs/telegram/telegram-bot.md](docs/telegram/telegram-bot.md). Остановить запущенного бота можно командой
`npm run telegram:stop`.

## Конфигурация

Настройки задаются переменными окружения в файле `.env` (за основу возьмите `.env.example`). Минимально для запуска
нужны строка подключения к базе (`DATABASE_URL`) и ключ доступа к языковой модели (`OPENAI_API_KEY`); для канала
Telegram — ещё и `TELEGRAM_API_KEY`. Полный справочник всех переменных по группам, с назначением и значениями по
умолчанию, вынесен в отдельный документ [readme-docs/configuration.md](./readme-docs/configuration.md).

## Голосовой ввод и вывод

Бот умеет распознавать голосовые сообщения и отвечать синтезированной речью. Голосовой ввод по умолчанию выключен
(`VOICE_INPUT_ENABLED=off`); провайдером распознавания может быть AssemblyAI или Groq Whisper. Голосовой вывод также
выключен по умолчанию (`VOICE_OUTPUT_ENABLED=off`); при включении доступен выбор голоса (`VOICE_OUTPUT_VOICE`, например
`alloy` или `nova`), а длинные ответы перед озвучиванием сокращаются. Соответствующий код находится в каталоге
`src/voice/` (распознавание в `transcribe.js`, синтез в `tts.js`, перечень голосов в `voices.js`).

## Память и проактивность

Принцип памяти и пять её видов описаны в разделе [«Что умеет бот»](#что-умеет-бот). Подробный разбор выборки, записи и

памяти — в [docs/ai-bot-with-memory/07-secure-privacy.md](docs/ai-bot-with-memory/07-secure-privacy.md).

Проактивное расширение позволяет боту самому начинать разговор по уместному поводу: затянувшееся молчание, утреннее
приветствие, движение к цели, возвращение собеседника или внешнее событие. Все пороги и суточные лимиты настраиваются,
а по умолчанию режим выключен (`COMPANION_MODE=off`). Подробности — в
[docs/ai-bot-with-memory/09-proactivity.md](docs/ai-bot-with-memory/09-proactivity.md).

## MCP-инструменты

Бот может подключать внешние инструменты по протоколу Model Context Protocol (MCP). Клиент и конфигурация находятся в
`src/mcp/` (`client.js`, `config.js`), список серверов задаётся в файле `.mcp.json` (образец — `.mcp.json.example`).
Это позволяет расширять набор доступных модели инструментов без изменения кода самого бота.

## Разработка

Тесты используют встроенный тест-раннер Node.js и работают с реальной базой данных и моделями, поэтому требуют
заполненного `.env` и применённых миграций.

```bash
npm test          # многослойный набор проверок ядра (структура БД, память, приватность, диалог)
npm run check:llm # проверка доступности и скорости выбранной модели
```

Полный перечень проверок по подсистемам, служебных скриптов и раскладку исходного кода см. в
[readme-docs/development.md](./readme-docs/development.md).
