# Конфигурация

Полный справочник переменных окружения. Настройки задаются в файле `.env` (за основу возьмите `.env.example`, где
лежат те же ключи с комментариями). В главном `README.md` показана лишь компактная таблица самых важных ключей —
здесь приведён исчерпывающий перечень по группам с назначением и значением по умолчанию.

## Модели и провайдер

| Переменная | Назначение | По умолчанию |
|------------|------------|--------------|
| `OPENAI_API_KEY` | Ключ доступа к языковой модели. | — |
| `OPENAI_BASE_URL` | Адрес OpenAI-совместимого прокси (например, LiteLLM); если пусто — прямой OpenAI API. | пусто |
| `LLM_PROVIDER` | Выбор провайдера языковой модели. | `openai` |
| `MAIN_MODEL` | Основной агент: отвечает пользователю и вызывает инструменты. | `gpt-5.4-mini` |
| `AUX_MODEL` | Быстрая вспомогательная модель: классификация входящих сообщений. | `gpt-5.4-nano` |
| `EXTRACT_MODEL` | Модель извлечения фактов в память. | `gpt-5.4-mini` |
| `EMBED_MODEL` | Модель эмбеддингов для смыслового поиска (1536 измерений). | `text-embedding-3-small` |
| `LLM_STREAMING_ENABLED` | Потоковый вызов модели в ядре агента. | `on` |
| `CEREBRAS_API_KEY` | Ключ доступа к провайдеру Cerebras. | — |
| `CEREBRAS_BASE_URL` | Адрес API Cerebras. | — |
| `GROQ_API_KEY` | Ключ доступа к Groq (модели и распознавание речи Whisper). | — |

Модели проверялись скриптом `npm run check:llm` на OpenAI-совместимом прокси. Семейство `gpt-5.4-*` отвечает примерно
за 5–10 секунд; если нужен максимально быстрый отклик, задайте `MAIN_MODEL`/`AUX_MODEL`/`EXTRACT_MODEL=gpt-4o-mini`.

## База данных

| Переменная | Назначение | По умолчанию |
|------------|------------|--------------|
| `DATABASE_URL` | Серверная строка подключения к PostgreSQL (используется и для `CREATE DATABASE`). | `postgresql://postgres:1@localhost:5432/postgres` |
| `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USERNAME`, `DB_PASSWORD` | Отдельные части строки подключения — альтернатива `DATABASE_URL`. | — |
| `MEM_DATABASE_URL` | Прямая строка подключения к рабочей базе памяти. | производится из `DATABASE_URL` |
| `MEM_DB_NAME` | Имя отдельной базы данных для памяти агента. | `agent_mem` |

## Лимиты памяти

Сколько фактов каждой области попадает в запрос к модели. Список ранжируется по релевантности и обрезается до этих
значений.

| Переменная | Назначение | По умолчанию |
|------------|------------|--------------|
| `MEMORY_LIMIT_TOTAL` | Общий потолок числа фактов в промпте. | `30` |
| `MEMORY_LIMIT_PROFILE` | Устойчивые факты о пользователе и стиле общения. | `7` |
| `MEMORY_LIMIT_DIALOG` | Факты текущего диалога. | `5` |
| `MEMORY_LIMIT_DOMAIN` | Факты предметной области. | `12` |
| `MEMORY_LIMIT_REMINDER` | Активные напоминания. | `3` |
| `MEMORY_LIMIT_SECURE` | Обезличенные резюме защищённых данных. | `3` |
| `AUTH_SECRET` | Ключ шифрования защищённой памяти (AES-256-GCM). Обязательно смените в продакшене. | `dev-insecure-secret-change-me` |

## Поджатие истории

Последние `HISTORY_HOT_WINDOW` сообщений всегда передаются дословно; всё, что старше, сжимается в дайджест.

| Переменная | Назначение | По умолчанию |
|------------|------------|--------------|
| `HISTORY_COMPRESSION_ENABLED` | Сжатие старой части истории диалога. | `off` |
| `HISTORY_HOT_WINDOW` | Сколько последних сообщений не сжимать вообще. | `8` |
| `HISTORY_MAX_TOKENS` | Порог запуска сжатия холодной зоны (токены). | `2000` |
| `HISTORY_SHRINK_TOKENS` | Целевой максимум размера дайджеста (токены, должен быть меньше `HISTORY_MAX_TOKENS`). | `800` |
| `HISTORY_MIN_COMPRESS_GAIN` | Минимальный выигрыш сжатия, иначе дайджест не перезаписывается. | `0.35` |
| `HISTORY_SUMMARY_MODEL` | Модель построения дайджеста. | `gpt-5.4-nano` |
| `HISTORY_ZONE_WEIGHTS` | Веса ближней, средней и дальней зон истории. | `0.55,0.30,0.15` |
