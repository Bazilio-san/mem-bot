## Что сделано (по коммиту на задание)

- `9c20231` — Задание A. В строке ответа ассистента теперь хранится след сгенерированной картинки: компактный
  дескриптор без URL в `metadata.images`, а для ответа-«только картинка» (пустой текст модели) в `content`
  пишется человекочитаемый маркер `[Сгенерировано изображение: …]`. Возвращаемый каналу `answer` не меняется,
  поэтому доставка фотографии без текстовой заглушки работает по-прежнему. Файл `src/agent.js`.
- `d62748b` — Задание B. Факт, извлечённый из реакции пользователя, помечается обратной ссылкой
  (`reaction_target_message_id`, `reaction_key`) в своих метаданных; при снятии реакции эти и только эти факты
  архивируются с пометкой `revoked_by=reaction_removed`. Подтверждения независимых фактов и факты других реакций
  не затрагиваются; замена реакции (лайк на сердце) сохраняет факты новой реакции. Файлы `src/pipeline/facts.js`
  (параметр `opts.metadata` в `saveFact`/`saveFacts`, новая функция `revokeReactionFacts`) и `src/agent.js`.
- `25e05ee` — Задание C. Введён компактный след вызванных инструментов `[{name, ok}]` в `metadata.tools` ответа
  ассистента. При сборке истории для интент-детектора (`classifierHistory`) и для основной модели к строке
  ассистента добавляется префикс вида `[инструменты: scheduler_create_task ✓]`. Префикс ставится в начало строки,
  поэтому обрезка `truncateLine` в классификаторе (она режет хвост) его не теряет. Аргументы и URL не сохраняются —
  полный аудит остаётся в `mem.tool_calls`. Файл `src/agent.js`.
- `22b3602` — Задание D. Реакция бота записывается в историю самодостаточной строкой
  `Бот отреагировал :okay: на сообщение пользователя: «…»` (симметрично реакциям пользователя), а не сухим
  текстом-заместителем. Абстрактный токен сохраняет ядро независимым от конкретных эмодзи Telegram; текст-заместитель
  лежит в `metadata.fallback_text`. Файл `src/agent.js`.
- `84ac2ea` — Задание E. Когда целевое сообщение реакции не найдено по внешнему рефу, вместо вводящей в заблуждение
  пустой цитаты `«»` пишется `…на сообщение бота (текст сообщения недоступен)` и проставляется `target_unknown: true`
  в метаданных. Поведение при найденной цели не изменилось. Файл `src/agent.js`.
- `a9c07b0` — служебный коммит: форматирование `facts.js` под oxfmt.

## Решения, принятые по ходу (где план оставлял выбор)

- Задание B, жёсткость отзыва: выбран мягкий вариант — архивирование (`status='archived'`) с аудит-меткой, а не
  удаление и не снижение `confidence`. Архив убирает факт из активной выдачи, но сохраняет его для аудита истории.
- Задание B, область отзыва: обратная ссылка ставится только на факты, которые реакция РЕАЛЬНО создала (пути
  `created`/`replaced` в `saveFact`), но не на подтверждение уже существующего факта. Иначе снятие реакции ошибочно
  заархивировало бы факт из независимого источника.
- Задание D, форма эмодзи: в `content` пишется абстрактный токен (`:okay:`), а не глиф Telegram, чтобы не нарушить
  границу «ядро не знает про эмодзи конкретного канала». Реальный глиф при необходимости восстанавливается из
  `reaction_key` на слое канала.

Важная оговорка о покрытии: штатные наборы оценок подтверждают ОТСУТСТВИЕ РЕГРЕССИЙ, но напрямую не проверяют новое

рекомендуемый следующий шаг (см. ниже).
